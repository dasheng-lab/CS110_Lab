1.How do you pass command line arguments to a program when using gdb?

Answer: run $(arglist)

2.How do you set a breakpoint which only occurs when a set of conditions is true (e.g. when certain variables are a certain value)?

Answer: break ... if $(conditions)

    2.1. View all the breakpoints: info breakpoints
    2.2. Delete breakpoint(s): delete breakpoint 1 2 3 .../delete breakpoint 4-10

3.How do you execute the next line of C code in the program after stopping at a breakpoint?

Answer: n

4.f the next line of code is a function call, you'll execute the whole function call at once if you use your answer to #3. (If not, consider a different command for #3!) How do you tell GDB that you want to debug the code inside the function instead? (If you changed your answer to #3, then that answer is most likely now applicable here.)

Answer: s

    4.1 quit the function and back to the outer function: finish

5.How do you resume the program after stopping at a breakpoint?

Answer: c

6.How can you see the value of a variable (or even an expression like 1+2) in gdb?

Answer: print $(var)

7.How do you configure gdb so it prints the value of a variable after every step ?

Answer: display $(var)

    7.1. Undisplay auto printing of val: Undisplay $(seq-of-var)

8.How do you print a list of all variables and their values in the current function?

Answer: info locals

9. How do you exit out of gdb?

Answer:q
