# CS110 Labs & HWs & Projects
This is my Labs, homeworks and projects in course CS110.
## about CS110/CS110P
CS110 and CS110P are two core courses in Shanghaitech University CS major. CS110 is the course code of "Computer Architecture I", and CS110P is the course code of "Computer Aritecture I project".

CS110 worth for 4 credits, and corresponding CS110P worth 2 credits. These two courses are binding, which means you must take these two courses in one semester. CS110 concentrates more on academic knowledge about Memory Management, Assembly, OS and Computer structure, while the later focus more on practical usage of those knowledge to build a little meaningful gadget. You need a founded basis of C programming language to take these courses.

## about this repository
This repository will have only one branch except special cases. Excluding `.gitignore` files, this repository have several category of files/folders. I show those category by a python `re.search` representation.
### hw([0-9])+
Means homework (hw) folder. The number after "hw" represent the label (or order) of the homework. All the files related to this homework were put under this folder. These homework is more about some small application of the knowledge learned in class. The order of homework is smaller, the assignment released time and completed time earlier. The same as follow in lab and pj session.
### lab([0-9])+
Means lab folder. All the files related to this lab were put under this folder. These lab is more about learning fundamental software, online development tools, and some useful kits according to this course.
### pj[0-3]\.[0-3]
Means project folder. This is exactly one parts related to CS110P. All the files related to this session of projects were put under this folder. The "." means the session of this project. Project builds more complex, more challenging things, which is the hardest part. 
### \.vscode
Vscode configuration folders. Just ignore it. If you want to look through it, `setting.json` file may be useful to you.
### other file
Except this file and `.gitignore`, other files place there directly (i.e. not in any folder) is used for submission, don't need to consider it. If you indeed want to read it, go to the folder related to this file.
